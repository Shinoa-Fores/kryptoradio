Bitcoin Radio

Connect to the bitcoin network everywhere – even without the Internet!

View project onGitHub

General description

We are developing a project named Bitcoin Radio. Its goal is to transmit all bitcoin transactions, blocks, and currency exchange data real-time using Terrestrial television (DVB-T) transmitters around the world. Bitcoins in the air, literally speaking.

Any unidirectional digital transmission path with a sufficient error correction is suitable for this project. In addition to DVB-T there are many other possible ways to transmit Bitcoin stream like subcarriers of FM radio transmission, amateur radio, and DAB. We chose DVB-T for our pilot project because of its flexibility and wide support in most parts of the world.

The software part of this project consists of two separate applications: serializer and deserializer. The former connects to the the bitcoin network, receives transactions and blocks, then puts them into a queue, which is then processed according to given priorities and finally serialized to a stream of bytes. The result is passed to transmitter. On the other end, a tuner is connected to deserializer which parses the incoming stream and emulates a bitcoin node which serves transactions and blocks to all connected bitcoin nodes.

The data stream can contain other information, such as exchange rates between bitcoins and traditional currencies.

This scheme makes it easy to construct affordable receivers that do not need mobile data connections in order to follow bitcoin traffic an to react to the received bitcoin payments. This would make it possible to build bitcoin counterpart for cash payment terminals, anything from a cash register to a coin operated self-service laundry. If the receiver application follows only transactions relevant to itself, it could be possible to be build it even using an 8-bit microcontroller.

Also, it allows an alternative way access to Bitcoin network in cases where only very low speed Internet connection is available. When the block sizes grow in the future, this scales up. Handling of 10 megabyte blocks of the future is not an issue. And for every tin foil hat out there, this is a way to connect to Bitcoin network without a trace! You only need on-line accesss when you want to make transactions yourself.

Current status:

The software:

The application is programmed in C (gnu99) and it depends on Glib, LibSSL and Linux. It is licensed under GNU GPL v3. The sources are available from GitHub.

  • Serializer connects and stays in the bitcoin network.
  • Serializer knows how to send transactions and blocks to serial port.
  • Serializer optimizes the sent blocks by referencing to already sent transactions, and does not retransmit them in a new block (exactly as the bitcoin network normally operates).

Not yet finished:

  • Proper transaction prioritization. Currently vulnerable to penny flooding.
  • Serial traffic format needs rethinking and someone else's opinion.
  • Serial traffic signing. The receiver needs to be sure that the data has not been forged.

Not yet started:

  • Receiver applicaltion (deserializer).
  • Testing in real radio network.
  • Sending exchange rates.

Distribution contracts and legal questions

We are currently negotiating of a possibility to start test transmissions in Finland using Digita's DVB-T network. There should be no legal pitfalls in Finland because bitcoin is considered legal (though it has no currency status, yet).

The pilot stage could be executed during a couple of months at night time (distribution between midnight and 6 a.m.) and smaller distribution area, e.g. channel group E (see Digita's coverage map), until the system is stable enough to be used for 24 hours in the whole country. It would be possible to receive the pilot stage transmissions in most areas of Finland.

Because crowdfunding is difficult in Finland, the reward for financing the project could be an advertisement, which the startup would sell. The contributors would be mentioned in the web page but each contributor could create a message that would be randomly assigned to the radio transmitted data stream, weighted according to the amount of money contributed. Maybe something like this could work.
